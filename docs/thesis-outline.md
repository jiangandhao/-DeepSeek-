# 毕业设计论文大纲与技术亮点

> 题目:基于 DeepSeek 大模型的全流程健康管理系统设计与实现(以血糖管理为核心)

## 论文章节建议

### 第一章 绪论
- 研究背景:慢病(糖尿病)管理需求、AI 大模型在医疗健康领域的机遇。
- 国内外现状:健康管理 App、医疗大模型、RAG 检索增强、时序血糖预测研究综述。
- 研究内容与目标:构建可演示的全流程健康管理系统,首期落地血糖管理智能体。
- 论文组织结构。

### 第二章 相关技术
- 大模型与提示工程、RAG 检索增强生成原理。
- DeepSeek 模型与 OpenAI 兼容 API。
- 时间序列预测(Naive/滑动均值/线性趋势/Prophet/LSTM 对比)。
- 技术栈:Vue3、Spring Boot、FastAPI、MyBatis-Plus、JWT、Docker。

### 第三章 需求分析
- 功能需求:用户管理、血糖/饮食/运动数据管理、趋势可视化、预测预警、AI 智能体方案生成。
- 非功能需求:安全(认证/越权/脱敏)、性能(并发设计)、可维护性、可扩展性。
- 用例图、业务流程图。

### 第四章 系统设计
- 总体架构(三层 + AI 微服务,见 `architecture.md`)。
- 数据库设计(见 `database.md`,含 E-R 图)。
- 接口设计(见 `api.md`)。
- 血糖管理智能体编排设计:数据汇总 → RAG 检索 → Prompt 工程 → DeepSeek 生成 → 流式回传。
- 安全设计:JWT 鉴权链路、全局异常、隐私脱敏。

### 第五章 系统实现
- 前端实现:登录注册、记录管理、ECharts 趋势图、SSE 流式对话界面。
- 后端实现:认证、数据 CRUD、AI 编排网关(WebClient 透传 SSE)。
- AI 服务实现:DeepSeek 客户端、轻量 RAG 检索器(字符 bigram + TF-IDF)、预测与异常检测。
- 关键代码与流程截图。

### 第六章 实验与测试
- **预测模型对比实验**:运行 `ai-service/notebooks/glucose_forecast_experiment.py`,
  在合成数据上对比 Naive-Last / Naive-Mean / MovingAvg / LinearTrend 的 MAE/RMSE/MAPE。
  讨论:餐后周期性导致滑动均值优于纯线性趋势 → 引出引入周期/季节性建模的改进方向。
- **RAG 召回示例**:`/api/agent/retrieve` 给出查询-召回片段-相似度,验证检索相关性。
- **异常检测验证**:构造高/低血糖/波动样本,验证分级与提示正确性。
- **联邦学习对比实验**:运行 `ai-service/notebooks/federated_learning_demo.py`,
  对比 集中式 / FedAvg(数据不出域)/ 各端单训 的准确率,论证「数据不出域、模型聚合」价值
  (示例:集中式 0.85 ≈ FedAvg 0.85 > 各端单训 0.84)。
- **隐私脱敏验证**:设置手机号后,数据库存储为 AES 密文(如 `gMLmjMaX0Y5hjyZLJGmH1w==`),
  API 返回脱敏值 `138****5678`,佐证数据治理。
- **功能测试**:注册→录入→趋势→预测→预警→AI 方案→风险评估→体检预约→影像检测 全流程用例。
- **性能测试**:`deploy/loadtest.py` 并发压测(示例:GET /api/glucose 并发50 达 ~1014 QPS、
  p95 112ms、成功率 100%),结合架构论述支撑高并发设计;亦可用 JMeter 规范化(见 high-concurrency.md)。

### 第七章 总结与展望
- 工作总结、不足、未来工作(语义检索升级、真实数据集、影像识别、联邦学习落地)。

## 技术亮点论述(对应原始项目要求)

| 亮点 | 本系统的实现/论述 |
|---|---|
| DeepSeek 医疗领域适配 | 以**提示工程 + RAG 检索增强**实现领域适配(替代昂贵的真实微调),论文说明权衡:无需 GPU、可快速迭代、知识可更新 |
| 多模态数据融合 | 结构化指标(血糖/饮食/运动)统一进入智能体上下文;架构预留影像推理结果接入风险评估 |
| 隐私脱敏 | 敏感字段加密存储 + 前端按权限脱敏(真实实现,数据治理章节) |
| 联邦学习 | 架构论述 + 可选 Flower 最小 Demo(2 客户端模拟),讲清「数据不出域、模型聚合」原理 |
| 影像识别(肺结节) | 后续阶段用开源预训练模型做推理演示 + 报告可视化 |
| 千万级并发 | 架构方案:无状态服务 + LB + Redis 缓存 + 读写分离/分库分表(user_id 分片)+ MQ 削峰 + CDN;附小规模压测 |
| 医疗 AI 数据治理 | 数据脱敏、AI 决策可追溯(`ai_advice.context` 记录数据与检索依据) |

## 答辩演示脚本(建议)

1. 配置真实 `DEEPSEEK_API_KEY`,`docker compose up`,运行 `deploy/seed_demo.py` 导入演示数据。
2. 登录 → 概览页展示血糖趋势(正常区间/异常点标注)。
3. AI 智能体页:点击「生成综合方案」→ 流式输出个性化饮食+运动方案。
4. 提问「我餐后血糖偏高,怎么调整?」→ 展示结合数据 + 知识库的回答。
5. 展示预测面板(MAE/RMSE)与异常预警列表。
6. 运行预测对比实验脚本,展示模型评估表。
7. 讲解架构图与高并发/隐私设计。
